데이터 수집 파이프라인

1

원천 수집 — CSV, GPX, Shapefile

공공데이터포털·산림청·한국등산트레킹지원센터에서 다운로드. 인코딩: UTF-8 / CP949 혼재, 좌표계: EPSG:4326 / 5179 / 5186 혼재.

2

파싱 & 변환

CSV → pandas 파싱 (CP949 자동 감지). GPX → xml.etree.ElementTree (병렬 16스레드). Shapefile → geopandas → WGS84(EPSG:4326) 재투영 → GeoJSON.

3

속성 산출 — GPX 분석

Haversine 공식으로 총 거리 계산. 고도 데이터 있을 경우 누적 상승·하강고도 산출. Naismith 공식(4km/h + 고도 100m당 +30분)으로 예상 소요시간 계산.

4

정규화 & 통합

컬럼명 표준화 (한국어 → 스키마 필드명). 카테고리 ENUM 매핑. 지역명 정규화. UUID 부여.

5

DB 적재 & API 제공

courses / poi / geo_layers 테이블에 upsert. GeoJSON 파일은 정적 서빙 또는 PostGIS 적재. REST API: GET /courses, GET /courses/{id}/gpx, GET /layers/{id}.
